# 会员身份ID设置说明

## 配置会员角色ID

在 `.env` 文件中设置以下配置：

```bash
# Discord服务器ID
GUILD_ID=你的服务器ID

# 会员角色ID（申请体验时会自动分配此角色）
MEMBER_ROLE_ID=你的会员角色ID

# 管理员角色ID（多个用逗号分隔）
ADMIN_ROLE_IDS=管理员角色ID1,管理员角色ID2
```

## 如何获取角色ID

### 方法1：Discord开发者模式

1. 打开Discord设置 → 高级 → 开启"开发者模式"
2. 在服务器中，右键点击会员角色
3. 选择"复制ID"
4. 将ID粘贴到 `.env` 文件的 `MEMBER_ROLE_ID` 中

### 方法2：使用机器人命令

1. 在Discord服务器中，使用机器人命令查看角色列表
2. 或者使用Discord的开发者工具查看

## 配置示例

```bash
# 示例配置
GUILD_ID=1234567890123456789
MEMBER_ROLE_ID=9876543210987654321
ADMIN_ROLE_IDS=1111111111111111111,2222222222222222222
```

## 功能说明

- **MEMBER_ROLE_ID**: 用户申请体验权限时，会自动分配此角色
- **体验权限过期后**（6小时），会自动移除此角色
- **正式会员过期后**，也会自动移除此角色
- **GUILD_ID**: Discord服务器ID，用于识别服务器
- **ADMIN_ROLE_IDS**: 管理员角色ID列表，用于权限验证

## 注意事项

1. 确保机器人有"管理角色"权限
2. 确保会员角色的位置低于机器人的角色
3. 配置后需要重启机器人才能生效
